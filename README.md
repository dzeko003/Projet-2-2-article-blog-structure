# PROJET 2.2 : Article de Blog Structuré

## Description

Ce projet a été réalisé dans le cadre de la formation Akieni Academy – Phase 2 (HTML, CSS & Git).
Il consiste à créer une page d’article de blog complète et structurée en HTML sémantique, intégrant différentes sections typiques d’un blog moderne.

L’objectif principal est de maîtriser la structuration d’une page web en utilisant les balises HTML5 appropriées ainsi que les bonnes pratiques d’accessibilité et d’organisation du contenu.

---

## Objectif du projet

- Concevoir une page d’article de blog structurée
- Utiliser une sémantique HTML5 correcte
- Intégrer différents types de contenus (texte, images, tableaux, formulaires)
- Respecter les bonnes pratiques d’accessibilité
- Structurer une interface proche d’un blog réel

---

## Contenu de la page

La page `index.html` comprend les éléments suivants :

### En-tête de l’article

- Titre de l’article
- Auteur
- Date de publication
- Catégorie

### Corps de l’article

- Paragraphes structurés
- Citations (blockquote)
- Images avec légendes (figure / figcaption)
- Tableau de données intégré dans l’article

### Navigation et structure globale

- Menu de navigation (nav)
- Sidebar contenant des articles récents (aside)

### Section commentaires

- Formulaire complet de commentaires
- Champs de saisie structurés (nom, email, message, etc.)

---

## Technologies utilisées

- HTML5
- Structure sémantique (article, aside, nav, section)
- Gestion des formulaires HTML
- Tableaux HTML
- Accessibilité (alt, labels, structure logique)

---

## Critères de validation

- Utilisation correcte de la sémantique HTML5
- Présence d’un tableau bien structuré
- Formulaire de commentaire complet et fonctionnel
- Utilisation d’au moins 5 balises HTML différentes correctement
- Respect des bonnes pratiques d’accessibilité (alt text, labels)
