De zéro à Full-Stack depuis le Congo Mon parcours tech personnel

Par Berenis Publié le 30 juin 2026 Mis à jour le 30 juin 2026 Temps de lecture : environ 8 minutes

Tags : self-taught, full-stack, React, Laravel, Congo, carrière

Introduction

Il y a quatre ans, je ne savais pas ce qu'était une balise HTML. Aujourd'hui, je déploie des architectures microservices avec Docker, Traefik et GitHub Actions depuis Brazzaville, au Congo Brazzaville. Ce n'est pas une histoire de talent inné c'est une histoire de méthode, de constance et de curiosité.

Dans cet article, je vous raconte mon parcours : les ressources que j'ai utilisées, les erreurs que j'ai commises, les projets qui m'ont le plus appris, et les conseils que j'aurais voulu recevoir au départ.

1. Les premiers pas: HTML, CSS et la découverte du web

Tout a commencé par une simple question : «Comment les sites web sont-ils faits?». J'ai ouvert le code source d'une page dans mon navigateur, vu des centaines de balises incompréhensibles, et au lieu d'être découragé j'ai été fasciné.

Ma première ressource a été freeCodeCamp, entièrement gratuite. J'ai appris HTML en deux semaines, CSS en un mois. Mais la vraie leçon de cette période n'était pas technique c'était d'apprendre à apprendre.

«Do the difficult things while they are easy and do the great things while they are small. A journey of a thousand miles must begin with a single step.»

Lao Tzu , Philosophe chinois

cette citation est devenue ma philosophie : Faites les choses difficiles tant qu'elles sont faciles, et faites les grandes choses tant qu'elles sont encore modestes. Un voyage de mille li commence toujours par un premier pas.

2. Le saut vers JavaScriptet pourquoi c'est difficile pour tout le monde

JavaScript m'a demandé beaucoup plus de temps que HTML/CSS. Les concepts de closures, de promesses et de programmation asynchrone ne s'apprennent pas en une semaine.

ce qui m'a sauvé : contruire plusieurs projet , faire des exercices pratiques.Rien n'ancre un algorithme dans la mémoire mieux qu'un projet concret et amusant.

4. Les projets qui m'ont vraiment formé

Les tutoriels donnent la base. Les projets construisent le développeur. Voici les trois projets qui ont le plus accéléré ma progression

  1. Jodesse Photography Portfolio photo avec menu hamburger, header glassmorphism et bouton de chat flottant. Premier vrai projet
  2. Dosscopy Plateforme multi-services en production : microservices Laravel, dashboards React, Docker + Traefik, CI/CD GitHub Actions. Le projet qui m'a tout appris sur la production réelle.
  3. NdakoPlateforme immobilière pour Brazzaville, en cours de conception. Visites 360°, paiements Mobile Money, cartographie hyper-locale. Mon premier projet produit

«La meilleure façon d'apprendre à coder, c'est de coder quelque chose qui vous importe vraiment.»

Mon expérience personnelle

5. Mes conseils si vous débutez depuis l'Afrique

Apprendre à coder depuis Brazzaville, c'est possible. Voici ce que j'aurais voulu savoir dès le départ.

Commencez par HTML et CSS, pas par Python ou JavaScript
Le feedback visuel immédiat est crucial pour rester motivé. Vous voyez le résultat de votre code dans le navigateur en temps réel.
Utilisez Git dès le premier jour
Même si vous ne comprenez pas tout, prenez l'habitude de git add . && git commit -m "..." quotidiennement. Votre historique deviendra votre portfolio.
Construisez en local, deployez en ligne
Vercel et Railway offrent des tiers gratuits. Avoir une URL réelle à montrer change tout lors d'une recherche d'emploi.
Rejoignez des communautés en ligne
Discord, X (Twitter) tech, GitHub Issuesles communautés compensent l'absence de meetups locaux.
La régularité bat l'intensité
1 heure par jour pendant un an vaut mieux que 10 heures par semaine pendant deux mois puis rien.

Conclusion

Mon parcours n'est pas terminéil ne le sera jamais, parce que le développement web évolue constamment. Mais si je devais résumer en une phrase ce que j'ai appris:

«La géographie ne détermine plus votre potentiel en tech. Votre connexion internet et votre régularité, oui.»

— Berenis
